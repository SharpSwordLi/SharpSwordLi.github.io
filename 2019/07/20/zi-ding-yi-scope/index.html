自定义Scope
spring scope

spring中Scope一般分为singleton，prototype,request,session，在查阅资料时发现scope可以自行定义，根据资料自己也写了一个线程绑定的范围的Scope和大家分享下

一.Spring中Scope基础范围

  1. singleton: 在Spring的IoC容器中只存在一个对象实例（单例），所有该对象的引用都共享这个实例。（默认）
  2. prototype:每次对该bean的请求都会创建一个新的实例，一般情况下，有状态的bean使用该scope。
  3. request：每次http请求将会有各自的bean实例，类似于prototype。（在web下生效）
  4. session：在一个http session中，一个bean定义对应一个bean实例。（在web下生效）

二.自定义Scope

直接创建一个springboot项目进行测试

1.定义自定义Scoope类

package com.example.demo.scope;

import org.springframework.beans.factory.ObjectFactory;
import org.springframework.beans.factory.config.Scope;

import java.util.HashMap;
import java.util.Map;
//先定义一个类实现Scope接口，并实现Scope中的5个方法
public class LocalScope implements Scope {
    //定义一个ThreadLocal本地线程变量，里面给一个Map用于储存bean的名字以及对应的实例
  private final  ThreadLocal<Map<String,Object>> threadLocal = new ThreadLocal<Map<String, Object>>(){
      //内部类重写initialValue方法
      @Override
      protected Map<String, Object> initialValue() {
          return new HashMap<>();
      }
  };
  //用户获取该bean对象时会调用该方法
    @Override
    public Object get(String name, ObjectFactory<?> objectFactory) {
        //从线程中获取Map对象
        Map<String, Object> scope = threadLocal.get();
        //从Map中获取bean实例
        Object obj = scope.get(name);
        //如果为空从容器中取得该对象
        if (obj==null){
            obj = objectFactory.getObject();
            scope.put(name,obj);
            System.out.println("Not exists " + name + "; hashCode: " + obj.hashCode());
        } else {
            System.out.println("Exists " + name + "; hashCode: " + obj.hashCode());
        }
        return obj;
    }
    //底层移除name对应的对象。实现者需要同时移除注册的销毁化回调方法
    @Override
    public Object remove(String name) {
        Map<String, Object> scope = threadLocal.get();
        return scope.remove(name);
    }
    // 注册一个销毁时的回调方法
    @Override
    public void registerDestructionCallback(String name, Runnable callback) {

    }

    @Override
    public Object resolveContextualObject(String key) {
        return null;
    }

    @Override
    public String getConversationId() {
        return null;
    }
}

2.注册Scope

现在已经定义好自定义的Scope类，然后将它注册到spring配置中

package com.example.demo.scope;

import org.springframework.beans.factory.config.CustomScopeConfigurer;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
//交给spring
@Configuration
public class ScopeConfig {
    @Bean
    public CustomScopeConfigurer customScopeConfigurer(){
        //创建添加scope的配置类
        CustomScopeConfigurer customScopeConfigurer = new CustomScopeConfigurer();
        //新建一个本地Scope示例
        LocalScope localScope = new LocalScope();
        //添加scope “localScope”是调用时的属性名
        customScopeConfigurer.addScope("localScope",localScope);
        return customScopeConfigurer;
    }
}

3.定义一个bean对象

package com.example.demo;

import org.springframework.context.annotation.Scope;
import org.springframework.stereotype.Service;

@Service
@Scope("localScope")
public class TestService {
    public void getMessage(){
        System.out.println("HelloWorld");
    }
}

4.测试

现在spring中已经有我们的Scope了,直接在测试类里进行测试。

package com.example.demo;

import org.junit.Test;
import org.junit.runner.RunWith;
import org.springframework.beans.BeansException;
import org.springframework.beans.factory.BeanFactory;
import org.springframework.beans.factory.BeanFactoryAware;
import org.springframework.boot.test.context.SpringBootTest;
import org.springframework.test.context.junit4.SpringRunner;

@RunWith(SpringRunner.class)
@SpringBootTest
public class DemoApplicationTests implements BeanFactoryAware{
    //获取spring工厂
    private BeanFactory factory;
     @Override
    public void setBeanFactory(BeanFactory beanFactory) throws BeansException {
        this.factory = beanFactory;
    }

    @Test
    public void contextLoads() throws InterruptedException {
        //线程1
        Thread task1 = new Thread(){
            @Override
            public void run() {
                TestService service = factory.getBean("testService", TestService.class);
                TestService service1 = factory.getBean("testService", TestService.class);
                TestService service2 = factory.getBean("testService", TestService.class);
                service.getMessage();
                service1.getMessage();
                service2.getMessage();
            }
        };
        task1.start();
         //线程2
        Thread task2 = new Thread(){
            @Override
            public void run() {
                TestService service = factory.getBean("testService", TestService.class);
                TestService service1 = factory.getBean("testService", TestService.class);
                TestService service2 = factory.getBean("testService", TestService.class);
                service.getMessage();
                service1.getMessage();
                service2.getMessage();
            }
        };
        task2.start();
         //线程3
        Thread task3 = new Thread(){
            @Override
            public void run() {
                TestService service = factory.getBean("testService", TestService.class);
                TestService service1 = factory.getBean("testService", TestService.class);
                TestService service2 = factory.getBean("testService", TestService.class);
                service.getMessage();
                service1.getMessage();
                service2.getMessage();
            }
        };
        task3.start();
        //让主线程睡10s
        Thread.sleep(10000);
    }

}

用Junit测试多线程需要注意，如果主线程停止，其他线程也会停止，所以让主线程睡10s保证其他线程处理完毕，也可以通过join方法进行阻塞测试。
