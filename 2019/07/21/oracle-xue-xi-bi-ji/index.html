一.Orecle部分语法

1. SQL92 联合查询

将（+）放在左侧代表右外连接，放在右边代表左外连接

左外连接 

select 字段 from 表1,表2 where 表1.字段=表2.字段（+）

右外连接 

select 字段 from 表1,表2 where 表1.字段（+）=表2.字段

2. SQL99 联合查询

笛卡尔积：使用cross join

select 字段 from 表1 cross join 表2

自然连接：使用natural join 会自动筛选同名同值的字段。

select 字段 from 表1 natural join 表2

using关键字，进行筛选

select 字段 from 表1 inner join 表2 using（筛选字段,...）

on关键字，进行筛选

select 字段 from 表1 inner join 表2 on 条件 where 条件

左外连接查询

select 字段 from 表1 left join 表2 on 条件 where 条件

右外连接查询

select 字段 from 表1 right join 表2 on 条件 where 条件

全外连接查询

select 字段 from 表1 full join 表2 on 条件 where 条件

3. 子查询

  1. any 任意
  2. all 所有
  3. in 任意存在 相当于 = any
    select 字段 from 表 where 字段 比较运算符 any 子查询语句
select 字段 from 表 where 字段 比较运算符 all 子查询语句
select 字段 from 表 where 字段 in 子查询语句
select 字段 from 表 where 字段 not in 子查询语句

    4. 主键约束

    创表的同时添加主键约束，直接在表后面添加
    constraints pk_表名_字段名 primary key(字段名);
    创建表后添加删除主键约束（给主键起名方便报错后查找）
    alter table 表名 add constraints pk_表名_字段名 primary key(字段名);
alter table 表名 drop constraints 约束名;

    5. 非空约束

    创表的同时添加非空约束，直接在表后面添加
    constraints ck_表名_字段名 check(字段名 is not null);
    创建表后添加删除非空约束（给主键起名方便报错后查找）
    alter table 表名 add constraints ck_表名_字段名 check(字段名 is not null);
alter table 表名 drop constraints 约束名;

    6. 检查约束

    创表的同时添加非空约束，直接在表后面添加
    constraints ck_表名_字段名 check(约束条件);
    创建表后添加删除非空约束（给主键起名方便报错后查找）
    alter table 表名 add constraints ck_表名_字段名 check(约束条件);
alter table 表名 drop constraints 约束名;

    7.分页查询

    使用rownum关键字,rownum只能小于等于不能做大于判断
    m为每页条数，n为页数
    select 字段 from(select rownum r,字段 from 表 where rownum<=m*n) where r>m*(n-1)

    三.索引

特点：显示的创建，隐式的执行，oracle会为主键创建默认索引。

1.优点：

提高数据的查询性能

  1. 通过创建唯一性索引，可以保证数据库表中每一行数据的唯一性
  2. 可以加速表与表之间的连接，特别是在实现数据的参考完整性方面有特别的意义
  3. 在使用分组和查询自居进行数据检索时同样可以显著减少查询分组和排序的时间

2.缺点:

  1. 创建索引和维护索引需要消耗时间，时间随着数据量的增加而增加。
  2. 索引需要占用物理空间，如果建立聚簇索引，那么需要的空间就会更大
  3. 对表中数据进行增删改时，索引需要动态的维护，这样就降低了数据维护的速度

3.适合创建索引字段：

  1. 在经常搜索的列上创建索引，可以加快搜索速度
  2. 在作为主键的列上
  3. 在经常需要根据范围进行搜索的列上创建索引，因为索引已经排序，其指定的范围是连续的
  4. 在经常需要排序的列上创建索引
  5. 在经常使用where子句中的列上创建索引，加快判决速度

4.不适合创建索引字段

  1. 在查询中很少使用，或者参考的列不应该创建索引。

  2. 对于有很少数据值的列也不应该增加索引

  3. 对于定义为text，image和bit数据类型的列不应该增加索引。这是因为这些列要么数据量相当大，要么取值很少。

  4. 当修改性能远远大于检索性能时。

    5.索引类型

  5. B-Tree（balancetree）

  6. Full-text（针对like进行查询优化）

    6.创建索引

  7. 普通索引

    create index index_表名_字段名 on 表名(字段名（长度）)

    删除索引

    drop index 名字

    7.索引碎片整理

    当数据库运行几年时间后，期间对表进行增删改，都会导致索引的碎片化，这时就应该对表的索引进行分析整理。
    主要分析三个元素信息：

  8. HEIGHT >= 4(索引的高度)

  9. PCT_USED < 50%(为表内每个数据块维持的已用空间最低百分比)

  10. DEL_LF_ROWS/LF_ROWS > 0.2(索引碎片比率)
    ```sql

– 分析索引(分析后，分析表中才有数据)
analyze index 索引名 validate structure;
– 查看索引分析信息
select name,HEIGHT,PCT_USED,DEL_LF_ROWS/LF_ROWS from index_stats;
– 重构索引
alter index 索引名 rebuild [online] [tablespace name];

## 四.视图
### 1.创建视图（创建者需要DBA权限）  
特点1：保护真实表，隐藏重要字段数据，保护数据
特点2：在视图中的操作会映射执行到真实表中
特点3：可以手动开启只读模式
```sql
create view 名字 as select 对外提供的内容 from 真实表名 
create view 名字 as select 对外提供的内容 from 真实表名 with read only

2.创建物化视图

create materialized view 名字 [选项] as select 对外提供的内容 from 真实表名

选项包括5种

  1. BUILD[IMMEDIATE,DEFERRED] 是否生成数据
  2. REFRESH[FAST,COMPLETE,FORCE,NEVER]刷新策略
  3. ON[DEMAND,COMMIT]手工刷新，自动刷新
  4. START WITH 数据第一次复制时间
  5. NEXT 刷新时间间隔

    3.删除视图

    drop view 名字

    五.SQL优化

  6. 对查询进行优化，应尽量避免全表扫描，应考虑在where以及order by涉及的列上建立索引。
  7. 避免在where子句中对字段进行null值判断（null值不会添加索引），解决办法在字段给默认值。
  8. 避免在where子句中使用！=或<>操作符，以及or连接符,如：
    ```sql
    select id from t where num=10 or num=20

– 替换为
select id from t where num=10
union all
select id from t where num=20

4. 能用between and 就不要用 in，not in
5. 少用like模糊查询
6. 避免在where子句中对字段进行表达式操作
7. 避免在where子句使用函数对字段进行操作
8. 不要的在where子句中的“=”左边进行函数运算
9. 很多时候用exists代替in是一个好的选择
```sql
select num from a where num in(select num from b)
-- 替换为
select num from a where exists(select 1 from b where num = a.num)
  1. 一个表的索引最好不要超过6个
  2. 只包含数字信息的不要设计为字符型，数字对索引只需比较一次，而字符需要一个一个去比较
  3. 任何地方都不要使用 * ，用具体字段代替*
